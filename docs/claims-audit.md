# ZeroModel claims audit

This document compares the public ZeroModel claims against the current clean `zeromodel` package.

Sources reviewed:

- Public blog post: <https://programmer.ie/post/zeromodel/>
- Current package README
- Current implementation under `zeromodel/`
- Current tests under `tests/`

## Status labels

| Status | Meaning |
|---|---|
| **Validated** | Implemented and covered by explicit tests in this repository. |
| **Implemented / thin evidence** | Code exists, but tests are smoke-level or do not yet prove the stronger wording. |
| **Not validated** | The repo does not currently contain the benchmark, fixture, or reproducible evidence needed for the claim. |
| **Reframe** | The claim should be narrowed because the implementation supports a weaker, more precise version. |

## Current honest headline

The repo can honestly claim:

> ZeroModel turns scored data into deterministic, inspectable Visual Policy Map artifacts. For closed enumerable policies, those artifacts support optimized state-addressed lookup without model invocation, Q-bearing evidence metrics, exhaustive named row properties, exact counterexamples, linked verification lineage, and export to a dependency-free Lua consumer. In the committed closed arcade world, a separately identified visual index recovers exact quantized feature codewords, policy rows, and actions from canonical observations across all 112 states and 2,401 tested waves. It rejects the committed blank and feature-changing fixtures, and a separate synthetic fixture proves the ambiguity rule can fire. The current arcade evidence does not yet demonstrate accepted non-exact nearest-neighbour tolerance. The package now also defines and tests a provider-neutral visual-address seam, separate dense-tensor identity, prototype-to-policy bindings, deployment-pair identity, and family-held-out benchmark schemas. Those are contract mechanics, not evidence of frozen embeddings, learned perception, occlusion robustness, or visual generalization. For scored matrices, ZeroModel can recover planted row structure in a committed synthetic fixture, calibrate it against a selection-corrected permutation null, and materialize the report and discovered ordering as linked artifacts.

## Claims to avoid until benchmarks exist

Avoid or soften these phrases in README/package copy until the repository contains reproducible evidence:

- “planet-scale”
- “infinite memory”
- “constant-time decision”
- “milliseconds on tiny hardware”
- “25KB RAM”
- “survives image pipelines”
- “watch AI think” as a literal claim
- “true visual AI” as a general claim
- “images directly to solutions” in open worlds
- “perception without a model” for natural or unbounded scenes
- “tolerant visual recognition” from the current exact-codeword arcade evidence
- “frozen embedding addressing is implemented” until an encoder adapter and benchmark land
- “critical visual evidence is detected” from contracts or global embeddings alone
- “visual logic equals reasoning”
- “self-describing PNG” unless metadata chunks are implemented
- “understands why the model learned”
- “connects to every tracker automatically”
- “validated on real training runs” until sanitized real exports are added
- “detects hallucinations by itself”
- “automatically learns the best view for every task” until task benchmarks exist
- “discovers semantic policy regimes” from numeric seriation alone
- “detects anomalies” from the current pattern MVP
- “improves decision accuracy” except for narrow tested fixtures
- “improves human inspection” until a task study exists
- “finds the forty important steps in any dataset” until manifold benchmarks exist
- “formally verified policy” when only finite row-level properties were checked
- “VIPER criticality” for arbitrary action scores without Q-value or equivalent consequence semantics
- “safe policy” when only the published property set passed

## Recommended validation backlog

1. **Frozen visual-address benchmark** — compare the current deterministic reader, normalized template matching, frozen medoid prototypes, raw embedding k-NN, and a rejection-equipped linear probe under family-held-out benign variation, critical-evidence interventions, and OOD inputs. Report explicit false-acceptance and false-rejection denominators and operating curves.
2. **Governance-parity audit** — compare the full address/manifest/binding/report chain with a conventional model digest plus append-only JSON log on predeclared replay, version, candidate, tamper, and affected-decision questions. Reduce the architecture if the lightweight wrapper reaches parity at materially lower complexity.
3. **Fixed-camera visual-address benchmark** — test an installation-calibrated bounded scene and deterministic features versus a pinned learned extractor, with physical or temporal corroboration for any safety-relevant absence claim.
4. **Pattern detector benchmark** — measure family false-positive calibration, planted recovery across signal-to-noise levels, perturbation stability, runtime, and a conventional seriation/clustering baseline.
5. **Dynamic policy verification** — extend [C60] with transition-system properties and compare finite scans with an SMT-backed checker on the same fixture.
6. **Criticality-weighted representation benchmark** — test [C62] by comparing uniform and criticality-weighted allocation under a fixed artifact budget.
7. **Human inspection study** — compare source-order, criticality-first, pattern-discovered, table, and log views for counterexample detection time and accuracy.
8. **PNG metadata round-trip** — test [C49] by embedding manifest/provenance in PNG text chunks and proving decode returns the same artifact ID.
9. **Hierarchy traversal benchmark** — test [C25] and [C36] with a generated corpus, child pointers, resolver, traversal harness, and raw timing report.
10. **Named edge benchmark** — test the compiled consumer on constrained hardware with measured memory and latency distributions.
11. **Decision manifold benchmark** — test [C20], [C29], and [C30] against known synthetic change points and random/scalar baselines.
12. **Signed verification lineage** — extend [C46] by combining artifact and verification identities with Sigstore or in-toto attestations.
13. **Sanitized critic exports** — test [C21], [C42], and [C43] with real critic or RAG-verifier outputs checked into fixtures.
14. **Sanitized real tracker exports** — strengthen [C15], [C16], and [C28]–[C32] with exports from actual training runs.
15. **Examples package** — strengthen [C14], [C21], and [C42] with search-triage, safety-gate, anomaly, and code-review fixtures.
16. **Constrained-runtime portability** — extend the Lua result with a C, Rust, MicroPython, or embedded-Lua consumer over a committed policy artifact.

## Bottom line

The cleaned repo validates the core artifact abstraction, deterministic multi-view and spatial/temporal consumers, optimized closed-world policy lookup, exhaustive exact-codeword observation addressing for one closed renderable arcade world, a non-vacuous and reachable ambiguity rejection contract, committed distance-based refusal fixtures, a provider-neutral visual-address contract over the deterministic reader, separate content-addressed tensor and prototype-binding contracts, deployment-pair identity, family-held-out benchmark schemas, Q-bearing criticality and decision-margin evidence, exhaustive finite row-property checking, linked verification and repair lineage, standard-Lua policy portability, and selection-corrected planted row-structure recovery with complete report/view materialization. It does **not** validate accepted non-exact visual tolerance in the arcade fixture, a frozen visual encoder, learned visual generalization, critical-evidence detection, open-world perception, general image understanding, arbitrary rendering robustness, general formal verification, automatic repair, safety certification, semantic pattern interpretation, anomaly detection, real-world pattern validity, tiny-hardware performance, or improved human inspection.
